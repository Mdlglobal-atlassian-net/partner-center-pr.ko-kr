# <a name="billing-overview"></a>청구 개요

**적용 대상**

-  파트너 센터
-  CSP 프로그램의 파트너

제품, 솔루션 및 서비스 고객을 대신해 구매에 따라 요금이 청구 됩니다 다음 방법 중 하나 이상에서 이러한 구매:
-   [라이선스 기반 청구](#licensebasedbilling)

    제품 또는 라이선스를 필요로 하는 온라인 서비스를 구입 하는 경우 (라이선스 사용 현황을)에 없는 구입한 각 라이선스에 대 한 요금이 청구 됩니다. 한 달에 한 번 또는 일년에 한 번 청구 될 것인지를 선택할 수 있습니다. 비즈니스 요구 변화 하는 경우에 서로 간에 전환할 수 있으며 다시 수 있습니다. 
    
    월간 및 연간 청구에 대 한 자세한 내용은 참조는 대금 청구 [FAQ](https://docs.microsoft.com/en-us/partner-center/faq-about-new-billing-features)합니다.

-   [사용량 기반 요금 청구](#usagebasedbilling)

    Azure 구독과 같은 온라인 서비스를 구매 하는 경우 월간 사용 요금에 대 한 요금이 청구 됩니다. 만 월별 청구는 사용량 기반 제품 제공 됩니다. Azure와 같은 사용량 기반 서비스 사용량을 기준으로 하는 요금제 요금에 따라 요금이 청구 됩니다.

-   [일회성 요금 청구](#onetimebilling)

    Azure 예약 또는 다른 소프트웨어 구독을 구입 하는 경우 미리 설정된 된 용어에 대 한 미리 지불 합니다. 미리을 지불 하기 때문에 하나의 전체 합 산치에에서 요금이 청구 됩니다. 
    
월별로 결제 하 게 하 여 선택한 또는 사용량 기반 월별 청구 되는 제품을 구입한 경우에 월간 청구 날짜는 파트너 센터에서 CSP 계정을 만들 때 선택한 월의 일입니다. CSP 계정에 성공적으로 만든 후 Microsoft는 결제를 포함 하는 확인 전자 메일을 보냅니다. 를 만든 후에이 날짜를 변경할 수 없습니다. 

## <a name="pricing-and-invoicing"></a>가격 책정 및 청구
가격표는 매달 업데이트되므로 한 달 전에 확인할 수 있습니다. 라이선스 기준 가격은 구독 기간(일반적으로 구매 날짜부터 12개월) 동안 보장됩니다. 사용량 기준 가격은 월 단위로 변경될 수 있습니다. 

제품, 서비스 및 소프트웨어 구독에 대 한 가격을 갱신할 때 가격이 변경 될 수 있지만 구독 기간을 통해 보장 됩니다.

크레딧 또는 조정은 적용된 후 다음 청구 송장에 조정 및 크레딧이 후불로 표시됩니다.

파트너 센터의 청구 페이지에서 송장 및 조정 파일을 보고 다운로드할 수 있습니다. 월별 송장은 파트너 센터에서 선택한 청구 날짜 4일 이내에 제공됩니다.

## <a name="payment-terms"></a>지급 조건

결제 용어는 net 60 일입니다. 청구서 지불 해야 기한 날짜 (청구 날짜 이후 60 일) 또는 계정 청구서 됩니다 연체, CSP에 등록 영향을 줄 수입니다. 과거 비용을 지불 하는 경우 일시 중단 된 계정의 전체 기능을 다시 사용할 수 있습니다 due 크기입니다.

### <a name="tax"></a>Tax

하는 세금이 부과 기반 세부 정보를 (없습니다 고객)에 Microsoft와 청구 관계는. 계정 설정 프로세스 중에 또는 나중에 지원 요청을 제출 하 여 세금 ID를 제출할 수 있습니다. 다음 청구 주기에 변경 내용이 반영된 것을 확인할 수 있습니다.

-   유지 하면서 및 판매세 예외에 대 한 지원 요청을 통해 세금 문서를 제출 해야 합니다. 다음 청구 주기에 변경 내용 및 적절한 환불을 확인할 수 있습니다.

-   추가 되는 값에 대 한 (부가가치세) 예외 서비스 요청을 통해 (Microsoft에서 유효성 검사) VAT ID를 제출 해야 합니다. 다음 청구 주기에 변경 내용 및 적절한 환불을 확인할 수 있습니다.

로컬 세금 사무실 전화 또는 세금 advisor에서 세금 세부 정보 추가 찾을 수 있습니다.

## <a href="" id="licensebasedbilling"></a>라이선스 기반 청구

고객 대신 라이선스 기반 제품을 구매 하는 경우 월간 또는 연간 요금이 청구 될 수 있습니다. 청구 주기가 나중에 변경 하려는 경우 아래 절차를 따르십시오. 

월별 청구에서 연간 청구로 전환는 일반 청구 날짜에 맞게 월별 청구 되는 다양 한 구독이 있는 경우에 유용 합니다. 연간 청구에서 월간 청구로 전환 하는 것은 개별 고객의 청구 날짜가 조정에 유용 합니다. 

대금 청구 주기를 변경 하면 연간 용어는 대금 청구 주기를 변경한 날짜와 갱신 날짜 설정 된 반영 하도록 업데이트 됩니다. 

비즈니스 요구 사항이 변경 될 때마다 대금 청구 주기를 변경할 수 있습니다. 

### <a name="billing-rules-for-annual-billing"></a>연간 대금 청구에 대 한 청구 규칙

-   구독은 연간 단위로 이루어지며 자동 갱신됩니다.

-   청구는 연간 구독당 월간 결제 12회 또는 연간 결제 1회입니다.

-   라이선스 기준 서비스의 경우 이전 청구 기간 종료 시 라이선스의 수에 따라 다음 청구 기간에 대한 요금이 사전 청구됩니다.

-   라이선스 수 변경 시 후불로 청구/입금됩니다(라이선스-일수를 기준으로 비례하는 계산). 비례하는 계산에서는 [ROUND((ROUND(단가 * 수량 / 비례하는 달의 일수, 2) * 비례하는 일수) / 수량, 2) * 수량] 수식을 사용합니다.

-   지불 (없습니다 라이선스 프로 비전)을 판매 하는 라이선스에 대 한 요금이 청구 됩니다.

### <a name="to-change-the-billing-frequency-of-an-online-service"></a>온라인 서비스의 대금 청구 주기를 변경 하려면

1.  선택 **고객** 파트너 센터 메뉴 선택에서 고객 구독을 변경 하려면. 

2.  고객의 구독 페이지에서 변경 하려는 구독을 선택 합니다. 

3.  세부 정보 페이지의 아래 **대금 청구 주기**를 선택 **월간** 또는 **연간**합니다. 청구 주기가 뿐만 아니라 변경 되려고 할 구독 목록을 변경 하는 방법에 대 한 중요 한 정보를 사용 하 여 확인 페이지가 표시 됩니다. 

4.  선택 **확인** 변경, 또는 **취소** 실행을 취소 합니다. 

### <a name="adjustmentscreditscancellations"></a>조정/크레딧/취소

Microsoft는 라이선스 기반 서비스의 취소에 대 한 조기 종료 요금을 청구 하지 않습니다. 

라이선스 기준 서비스의 취소 크레딧은 중도 취소에 사용되지 않은 일수(와 위의 수식에 따른 라이선스 감소)에 비례합니다.

## <a href="" id="usagebasedbilling"></a>사용량 기반 요금 청구

일부 Microsoft 제품 및 서비스 사용 되는 서비스에 대해서만 청구 되는 "종 량 제" 요금 청구 모델을 사용 합니다. 예를 들어, Microsoft Azure는이 모델을 사용합니다. 

### <a name="billing-rules"></a>청구 규칙
-   구독-월 되며 새 유료 서비스 요금이 자동으로 갱신 합니다. 각 월 이전 달의 사용에 대 한 요금이 청구 됩니다.

-   데이터 통신 서비스 요금은 송장 주기 내에서 변경할 수 있습니다. 

    -   가격 상승: 30 일 통지를 제공 됩니다.

    -   가격 하락: 변동 기간(일)을 반영합니다.

    -   기존 구독은 청구서 주기 시작 시 유효한 요금을 사용합니다.

    -   새 구독의 경우, 동일한 청구 주기 이내 만들 때 만들면서 날짜에 적용 속도 사용 합니다. 

### <a name="adjustmentscreditscancellations"></a>조정/크레딧/취소

다음 월별 청구 송장에 조정이 있는 결제가 표시됩니다.

Microsoft에서 사용량 기반 서비스의 취소에 대 한 조기 종료 요금을 청구 하지 않습니다. 

다음 월별 청구 송장에 SLA 크레딧을 비롯한 모든 유형의 크레딧이 표시됩니다.

## <a href="" id="onetimebilling"></a>일회성 요금 청구

1 년 또는 3 년 용어에 대 한 소프트웨어 구독 및 Azure 예약을 사전 구매할 수 있습니다. 사전에 구입 하면 하나 총액에 전체 비용에 대 한 요금이 청구 됩니다. 이 유형의 청구 일회성 청구를 라고 합니다.

>[!IMPORTANT]
>Azure 예약 및/또는 사용자에서 다른 통화를 사용 하 여 위치에 고객에 대 한 구독 소프트웨어를 구입한 경우 청구 통화 기본 고객의 위치에 위치가 아니라을 기반으로 합니다. 고객은 여러 위치에 있는, 별도 청구서를 받게를 각 통화 고객에 대 한 조정 파일의 요금이 부과 하는 데 필요한 적합 한 통화에서 고객 청구서 수 있습니다. 

### <a name="manage-your-one-time-billing"></a>일회성 청구 관리

**현재 청구 상태, 송장 등 여러 파일 보기**

1.  파트너 센터에서 선택 **청구** 차례로 **번** 청구 상태를 볼 수 있습니다. 

2.  자세한 정보를 보려면 송장 또는 조정 파일을 선택합니다. 

**고객의 주문 기록 보기**

1.  선택 **고객** 파트너 센터 메뉴에서.

2.  **고객** 페이지에서 주문 내역을 확인하려는 고객을 찾고 아래쪽 화살표를 선택하여 고객의 기록을 펼쳐봅니다. 

3.  **주문 보기**를 선택하여 주문 내역을 표시합니다.

**크레딧 메모를 다운로드 합니다.**

크레딧 또는 rebill 요청 해야 할 경우 크레딧 메모 원래 송장을 취소를 제공 해 드리겠습니다. 다음과 같은 이유로 신용/rebill를 요청할 수 있습니다.

-   주문 수정 처리 또는 구입

-   송장이 생성되고 세금 환불이 적용되었습니다. 원래 송장에 세금 환불이 다시 적용되도록 크레딧/재청구를 요청할 수 있습니다. 환불에 대해서도 마찬가지로 원래 송장에 대한 크레딧/재청구를 요청하고 환불을 적용할 수 있습니다.
