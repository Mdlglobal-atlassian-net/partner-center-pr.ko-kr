# <a name="transition-from-cloud-solution-provider-csp-direct-bill-partner-to-csp-indirect-reseller"></a>클라우드 솔루션 공급자 (CSP) 직접 청구 파트너에서 CSP 간접 리셀러로 전환 

>[!Note] 
>이 문서에서는 간접 재판매인 전환 하기로 결정 했으면 직접 청구 파트너에 대 한 것입니다. 

기존 직접 청구 테 넌 트를 사용 하 여 간접 재판매자 프로그램에 등록 할 수 있습니다. 

## <a name="get-started"></a>시작

1. 확인 파트너 프로필에 파트너 센터에서 한 MPN ID는 현재 합니다.

2. 로그인 파트너 센터에 전역 관리자로 간접 재판매자를 전환 하는 테 넌 트 직접 청구 합니다.

![개요](images/direct/direct1.png) 

3. 등록 양식의 파트너 세부 정보를 검토 합니다.

![지금 등록](images/direct/direct2a.png) 

4. 선택 지금 등록 합니다. 간접 재판매자 비즈니스 직접 귀사에 사용할 동일한 AAD 테 넌 트를 사용 합니다.

>[!Note]
>처음에이 새 전환 기능 년 12 월 1 주년 날짜 년 9 월을 사용 하 여 파트너를 위한 사용할 수 있습니다. "T가 9 월과 년 12 월 간에 연주기,이 이번에는 기능을 표시 되지 않습니다 않는 경우. 2018 년 12 월 이후에 연주기 일 날짜를 사용 하 여 파트너는 파트너를 위한 기능을 사용 하도록 설정 하는 나중에 알림을 받게 됩니다.


5. 귀하의 등록이 승인 되 면 파트너 센터에 다시 로그인 합니다.

>[!Note]
> 승인 일반적으로 즉각적인 이지만, 영업일 기준 최대 5 개까지 걸릴 수 있습니다. 승인 되 면 지정한 등록 형태로 주 연락처에서 전자 메일 주소로 알림을 받게 됩니다. 등록 상태를 확인할 수도 있습니다 **설정을** → **파트너 설정을** → **파트너 프로필** → 프로그램 정보. 

6. 사용자 **개요** 페이지 간접 재판매인 계약 표시 됩니다. **동의 및 계속**을 선택합니다. 그러면 간접 재판매자 기능.

간접 재판매자 규약에 동의 하는 경우 파트너 프로필으로 식별 하는 공지 **둘 다** 간접 재판매자를 직접 청구 합니다.

![간접 재판매인 계약](images/direct/direct3.png)

>[!Important] 
>새 기능을 사용 하는 간접 재판매인으로 등록 하면 직접 청구-유일한 테 넌 트를 롤백할 수 있는 옵션이 있습니다. 간접 재판매인으로 등록 하기 전에 비즈니스 요구를 완벽 하 게 평가 해야 합니다.

## <a name="while-you-transition-from-direct-to-indirect-reseller"></a>간접 재판매자를 직접에서 전환 하는 동안

이 단계를 계속 관리 하는 동안 직접 고객의 구독 청구 프로세스를 포함 해야 합니다. 간접 공급자와로 작동 간접 재판매자의 고객을 받아들이지 시작할 수 있습니다.

![청구서를 직접 및 간접 재판매자는](images/direct/direct4.png)

## <a name="find-an-indirect-provider"></a>간접 공급자 찾기

를 등록 한 후에 왼쪽된 탐색에서 간접 공급자에 대 한 링크가 나타납니다. 간접 재판매자를 다음 결제 처리, 고객에 게 제품을 구매 하 고 수 있는 인프라를 지원 합니다. 간접 공급자를 사용 하 여 관계를 설정 합니다.

다양한 간접 공급자가 다양한 지원과 서비스를 제공하므로 간접 재판매인은 해당 지역의 공급자를 평가하여 요구 사항을 가장 잘 만족하는 공급자를 선정하면 됩니다. 일반적으로 대부분 공급자는 다음과 같은 일을 합니다. 

- 간접 재판매인에게 기술 지원 제공
- 제품 및 서비스 마케팅 지원 
- 의료 보험 재정 및 크레딧 사용 약관이 관리 

공식 목록 검색 [Microsoft 간접 공급자](https://partnercenter.microsoft.com/partner/find-a-provider)합니다.

자세한 내용은 [간접 공급자를 사용 하 여 파트너](indirect-reseller-tasks-in-partner-center.md)

## <a name="accept-a-partnership-invitation-from-your-indirect-provider"></a>간접 공급자의 파트너 관계 초대를 수락

통해 파트너는 간접 공급자를 찾아야 하는 경우 파트너 센터에서 간접 공급자를 사용 하 여 파트너 관계를 설정 합니다.

간접 공급자 선택 보내 전자 메일에서 파트너 센터에서 초대를 사용 하는 파트너 관계 초대 링크. 전역 관리자는 파트너 센터에 로그인 하 고 초대 링크 해야 합니다. 초대를 수락 하면 공급자의 이름을 간접 공급자 목록에 표시 됩니다.

## <a name="acquire-new-customers-as-indirect-reseller"></a>간접 재판매인으로 신규 고객 확보

및 간접 공급자 고객과 대리점 관계 포함 해야 합니다. 이 통해 고객의 구독 및 본인을 대신해 서비스를 관리할 수 있습니다. 기존 Azure AD 테 넌 트에 새 고객을 얻으려면 고객 및 공급자를 사용 하 여 동시에 형성 대리점에 초대할 수 있습니다. 

간접 재판매자 초대를 만들려면:

1. 선택 **간접 공급자** 파트너 센터에서 왼쪽 탐색 

2. **새 고객 초대**를 선택하여 귀사 및 간접 공급자와 동시에 재판매인 관계를 맺도록 고객을 초대합니다. 공급자, 고객과 대리점 관계 있으므로 고객이 새 구독을 구입 하거나 기존 구독에 새 라이선스를 추가 하려는 경우 대신 고객의 주문을 전송할 수 있도록 해야 합니다.

3. 다음 페이지에서 임시 전자 메일 메시지를 검토합니다. 전자 메일에 임시 메시지를 열 수 있습니다 또는 메시지를 클립보드에 복사 하는 전자 메일에 붙여 넣습니다.

4. 어떤 필요 하지만 고객이 직접 연결할 모두 사용자 계정 및 해당 공급자의 개인화 된 링크를 포함 해야 합니다. 예를 들어 전자 메일의 텍스트를 편집 합니다. 선택한 **수행**합니다.

5. 고객이 귀사와 공급자를 공식 재판매인으로 승인하면 귀사에는 고객을 대신하여 고객의 구독, 라이선스 및 사용자를 관리할 관리자 권한이 부여되고 간접 공급자는 고객을 대신하여 주문을 제출할 수 있게 됩니다.
6. 고객의 계정, 서비스, 사용자 및 라이선스를 관리하려면 고객 이름 옆에 있는 아래쪽 화살표를 선택하여 고객의 기록을 확장합니다. 

직접 청구 파트너와는 달리 간접 재판매인 파트너 센터에서 새 고객을 위해 Azure AD 테 넌 트를 만들 수 없습니다. 공급자에 게 테 넌 트 만들고이 고객에 대 한 간접 재판매자를 지정 합니다. 이 하면 고객이 파트너 센터에서 고객 목록에 나타납니다.

>[!Note]
>간접 재판매자를 구입한 고객에 대 한 구매를 만들려면에 직접 청구 기능을 사용할 수 없습니다.

1. 선택 **간접 공급자** 파트너 센터에서 왼쪽 탐색

## <a name="managing-your-direct-bill-customers-and-your-indirect-reseller-customers"></a>간접 대리점 고객 및 고객에 게 직접 청구 관리

간접 대리점 고객 및 고객에 게 직접 청구 다르게 관리 합니다. 

**Direct 청구서 고객만 (작업 간접 재판매자도 수행 되지 않습니다.)**

- 제품에 대 한 주문 만들기
- Azure 예약 관리
- 해당 주문 기록 관리
- 소프트웨어를 구입
- 고객에 게 직접 청구

**간접 재판매자의 고객은**

- 간접 공급자 고객에 게 제품을 주문
- 고객의 라이선스 및 사용자 관리
- 구독 갱신을 처리 합니다.

**직접 청구 파트너로 획득 한 고객을 식별**

1. 선택 **고객**

2. 해당 세부 정보를 보려면 고객을 선택

3. 옵션을이 고객을 직접 청구 파트너로 구입한 중 하나인 경우 나타납니다 **추가할** 또는 **제품 보기** 해당 구독을 볼 수 있습니다.

4. 고객에는 간접 재판매인 관계와 이러한 옵션은 사용할 수 없습니다.

## <a name="what-happens-when-your-direct-bill-capabilities-have-been-restricted"></a>직접 청구 기능에 제한 된 경우

청구서 및 간접 재판매자 모두 직접 또는 간접 재판매인으로 등록 하는 명시적 의사 결정을 수행 하지 않은 간접 재판매자 전환 있고 이중 등록 하기로 합니다 있는지 여부를 직접 새 요구 사항에 맞지 않는 청구서 파트너 CSP에 대 한 직접 청구 파트너 프로그램 때 알림을 받게 됩니다 Microsoft에서 직접 청구 기능과 제한 됩니다. 

사용자 **파트너 프로필** 아래에서 **정보를 프로그래밍** 에 Microsoft Cloud Solution Provider 상태가으로 변경 되는 표시 됩니다 **제한**합니다.

## <a name="affected-direct-bill-capabilities"></a>직접 청구 기능에 영향을

직접 청구 기능에 제한 된, 하는 경우 파트너 센터에서 고객에 대 한 새 구매를 더 이상 만들 수 없습니다. 다음을 포함합니다. 

- Azure 구독
- 라이선스 기반 구독
- 기존 라이선스 기반 구독에 새 추가 기능 추가
- 일회성 소프트웨어 구독, 영구 소프트웨어 및 Azure 예약 가상 머신 인스턴스 등의 소프트웨어 및 예약 제품 구매를 수행합니다.

또한 CSP 프로그램에서 Azure 파트너 공유 서비스 제품을 사용 하 여 사용자 고유의 사용에 대 한 새 Azure 구독 구입할 수 없습니다. 

신규 구매 하는 것 외에도 파트너 센터에서 다음 직접 청구 기능을 액세스할 수 없습니다. 
- 새 고객 테 넌 트를 만들 수 없습니다. 합니다 **만들기 고객** 옵션 **고객** 파트너 센터에 표시 되지 것입니다. 

- 직접 대리점 관계 요청 하는 고객에 게 초대를 생성할 수 없습니다. 합니다 **대리점 관계 요청** 옵션 **고객** 파트너 센터에 표시 되지 것입니다. 

>[!Note] 
>에 직접 청구 파트너 테 넌 트 간접 재판매자를 이미 등록 하는 경우에 직접 청구 하는 대신 간접 재판매인 관계를 요청 하는 고객에 게 초대를 보낼 수 있습니다. 

- 새 샌드박스 테 넌 트를 만들 수 없습니다. 각 직접 청구 파트너 테 넌 트 직접 청구 API 통합을 위해 하나의 샌드박스 테 넌 트를 만들 수 있습니다. 이전에 만들지 않은, 경우 청구 파트너 기능이 제한 되었습니다.를 직접 후 사용자에 게 이렇게 하려면 허용 되지 않습니다. 

## <a name="whats-not-affected"></a>어떤 영향을 받지 않습니다.

기존 구독을 직접 자재 명세서를 받지 않습니다. 유효한 상태를 유지 하며 자동으로 갱신 됩니다. 계속 하면 취소 될 때까지 Microsoft에서 직접 청구 됩니다. 

다음과 같은 방법으로 기존 구독을 관리 하는 계속 진행 합니다. 

- 기존 구독을 일시 중단 

- 기존 라이선스 기반 구독 사용자 수를 조정 합니다. 

- 구독에 기존 추가 기능 사용자 수를 조정 합니다. 참고: 추가 기능 신규 구매 취급 됩니다 때문에 기존 구독에 새 추가 기능을 추가할 수 없습니다. 

- 새 Azure 리소스를 배포 하 고 기존 Azure 구독에서 기존 Azure 리소스를 관리 합니다. 이 Azure marketplace 및 Visual Studio 구독을 통해 사용할 수 있는 제품을 포함 합니다. 

## <a name="enroll-for-indirect-reseller-incentives"></a>간접 대리점 인센티브에 대 한 등록 

기존 직접 청구 파트너 테 넌 트에서 간접 재판매인으로 등록 했습니다. 한 후 30 일 이내 간접 대리점 인센티브를 등록 하려면 초대를 받게 됩니다. 초대는 현재 CSP 파트너 테 넌 트와 연결 된 MPN 계정 파트너를 기반으로 합니다. 초대는 MPN 계정 파트너와 연결 된 전자 메일 주소로 보내집니다. 

동일한 파트너 테 넌 트를 사용 하 여 직접 청구 인센티브 프로그램에 대 한 등록도 가능 합니다. 프로그램을 개별적으로 관리 해야 합니다.

참고 항목 [간접 재판매자 점점 대 한 자세한 내용은](https://assetsprod.microsoft.com/csp-directbill-to-indirect-transition.pdf) 하며 [간접 재판매자 직접 청구 fAQ](https://assetsprod.microsoft.com/mpn/direct-bill-partner-faq.pdf).
