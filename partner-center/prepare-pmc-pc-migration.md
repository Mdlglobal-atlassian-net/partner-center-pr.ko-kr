# <a name="prepare-for-your-move-from-partner-membership-center-pmc-to-partner-center"></a>파트너 센터에서 파트너 멤버 자격 Center (PMC)을 이동에 대 한 준비

에서는를 이동 하는 멤버 자격 관리 파트너 멤버 자격 Center (PMC)에서 파트너 센터-Microsoft와의 비즈니스 관계를 관리 하는 단일 대상입니다. 파트너 센터는으로 효율성과 최대한 쉽게로 전환을 하려고 합니다. PMC에서 다른 파트너 센터 하 고 이해 하 고 이동 하기 전에에 준비 해야 하는 것이 생각 있는 몇 가지 영역으로 확인 되었습니다.

## <a name="account-and-identity-setup"></a>계정 및 id 설치

**Azure Active Directory (Azure AD) 작업 계정 이란?**

어떤 회사에서 Azure, Microsoft Intune, Office 365 같은 Microsoft 클라우드 서비스를 구독하면 Azure 공용 클라우드에 그 회사만을 위해 격리된 전용 가상 공간이 생성되는데, 이것이 바로 Azure 회사 계정입니다.

회사 계정을 Azure AD 사용자와 있으며 해당 전자 메일, 암호, 프로필 데이터, 권한 및 등에 대 한 정보를 호스트합니다. 회사 계정에는 또한 그룹, 응용 프로그램 및 회사 및 보안에 관련 된 기타 정보도 포함 되어 있습니다. 자세한 내용은 참조 하는 중...

파트너 센터에서 하지 개인 전자 메일 계정에 로그인 하 고 회사 전자 메일을 사용 합니다.
- 회사 계정: john@contoso.com
- 개인 계정: John@outlook.com

업무용 메일에는 Azure active directory 테 넌 트의 일부입니다. 파트너 센터에서 계정이, AAD 테 넌 트가 해야 합니다. Azure Active Directory에 대 한 자세한 내용은 읽을 [Azure AD에서 디렉터리를 만들](https://docs.microsoft.com/en-us/azure/active-directory/fundamentals/add-custom-domain#create-your-directory-in-azure-ad)합니다.

**으로 이동 하면 파트너 센터 PMC에서 어떤 계정을 해야 로그인 할 파트너 센터 경우 있습니다 (예: Office 365) 용 Microsoft으로 AAD 테 넌 트도가 CSP 비즈니스에 대 한 테 넌 트?**

CSP 계정 또는 MPN 회사 메일 계정을 사용 하 여 파트너 센터에 로그인 할 수 있습니다. CSP 업무용 메일을 사용 하 여 로그인 하려는 경우 CSP 및 MPN 프로그램 정보 대시보드에서 왼쪽된 탐색 창에 표시 됩니다. MPN Azure AD 테 넌 트 회사 전자 메일에 로그인 하면 MPN 프로그램 정보만을 표시 됩니다. 사용자 역할 MPN와 CSP 간의 다 MPN와 CSP 비즈니스에 대 한 동일한 계정을 사용 하는 경우 반드시 적절 하 게 사용자 역할을 할당 합니다. 사용자 역할에 대 한 자세한 내용은 [사용자 역할과 권한을 할당할](permissions-overview.md)합니다.

**AAD 전역 관리자 역할과 PMC MPN 전역 관리자 역할의 차이 무엇입니까?**

이들은 다른 사용 권한 사용 하 여 두 개의 완전히 다른 역할입니다. 파트너 센터에서 AAD 테 넌 트 전역 관리자 테 넌 트를 관리 하는 방법과 – 추가 또는 사용자를 제거, 제공 및 관리 암호, 역할 및 권한과 하 고 파트너 센터에서 회사의 모든 프로그램에 대 한 액세스 권한이 있습니다. 

PMC에서 MPN 전역 관리자 역할 다음을 수행할 수 있습니다.

- 회사 및 모든 회사의 위치와 관련 된 모든 데이터 보기 및 편집

-  로컬 또는 전역 수준에서 관리자를 추가 합니다.  또한 전역 관리자는 전역 관리자 액세스는 위치에 관계 없이 사용 하 여 연결 하는 전역 액세스 권한을 부여 하는 모든 위치에서 모든 사용자를 할당할 수 있습니다.
-  모든 파트너 연결 UI 함수 등을 수행 합니다. 

-  사용자를 추가/제거 합니다.

 - 역할 할당/제거 

 - 추가/제거/업데이트 위치 

 - 구매 역량/매핑 

-  혜택 보기

역할은 파트너 센터 전역 관리자 보다 작업과 다른 권한 있는 MPN 파트너 관리자 라고 MPN 전역 관리자가 파트너 센터를 이동할 때 에 대 한 자세한 내용은 파트너 센터에서 역할 및 권한을, [사용자 역할 및 사용 권한 할당](permissions-overview.md)

**파트너 센터에서 사용자 역할**

파트너 센터에 수행 해야 하는 데 필요한 작업의 형식에 따라 역할의 다른 형식이 있습니다. Azure AD 역할이 전역 관리자와 같은 역할 수 있습니다. 역할의 일부는 클라우드 서비스 공급자 프로그램 등 인센티브 프로그램에 특정 되며 MPN에 관련 된 역할입니다. 모든 파트너 센터 역할 이란를 찾으려면 읽을 [사용자 역할 및 사용 권한을 할당할](permissions-overview.md)합니다.

**파트너 센터 PMC에서 이동 하면 사용자의 역할에 됩니까?**

PMC에서 모든 사용자는 MPN 전역 관리자 또는 마이그레이션을 수행 하는 기본 프로그램 연락처를 제외 하 고 관리자 역할과 손실 됩니다. 마이그레이션을 완료 하는 개별 파트너 센터의 역할을 할당 해야 합니다. 파트너 센터에서 역할 PMC에서 다릅니다. 읽기 [사용자 역할 및 사용 권한을 할당할](permissions-overview) 하 고 [PMC에서 파트너 센터로 이동](https://docs.microsoft.com/en-us/partner-center/move-pmc-pc-map#user-roles) 파트너 센터에서에서 더 많은 사용자 역할에 대 한 합니다.


**내 회사 프로필 및 비즈니스 프로필 간의 차이 무엇입니까?**

회사 프로필에는 주소, 위치, 기본 연락처, 은행 및 세금 세부 정보를 포함 하는 회사에 대 한 정보입니다.

비즈니스 프로필에는 직접 고객에 게 이며 로고도 비즈니스 집중, 전문, 등의 세부 정보를 표시 하는 마케팅 페이지입니다.

**내 계정에 대 한 평균 통합 계정는 무엇입니까?**

동일한 Azure AD 테 넌 트를 사용 하 여 파트너 센터에 여러 MPN 계정을 마이그레이션할 경우 시스템 자동으로 인식 하를 계정의 통합 하도록 요청 합니다. 동일한 Azure AD 테 넌 트에 연결 된 여러 도메인이 있는 경우에 마찬가지입니다. 

파트너 센터를 마이그레이션 결정도 별도 AAD 테 넌 트를 사용 하 여 수 있지만이 인해 격리 평가 역량 및 추가 구매 비용을 유의 하십시오. 

**여러 AAD 테 넌 트 및 단일 MPN 계정에 있는 경우를 파트너 센터에 연결할 수 있습니까?**

예, 파트너 센터에서 Azure AD 테 넌 트를 여러 단일 파트너 센터 계정에 연결할 수 있습니다.
여기에서 알아보세요. 


## <a name="microsoft-partner-network-mpn-membership-migration"></a>Microsoft 파트너 네트워크 (MPN) 멤버 자격 마이그레이션 

**파트너 센터로 이동 하는 PMC에서 수행할 수 있는 사람?**

회사 MPN 전역 관리자 또는 주 프로그램 연락처 (이러한 두 역할을 동일한 사용자가 보유 종종) 시작 하 고 이동을 수행 수 있습니다.

**마이그레이션을 완료 하는 사용자 회사 법적 프로필 파트너 센터에서 기본 연락처 될?**

하지만 반드시, 기본 연락처 해야 계약서에 서명 하려면 권한 부여를 가진 사람입니다.

**Microsoft 내 내 MPN 멤버 자격을 마이그레이션할 수 있습니까?**

아니요. Microsoft은 파트너 센터에 멤버 자격 계정을 이동할 수 없습니다. PMC에 마이그레이션 프로세스를 시작 하려면 회사 계정 (로그인 자격 증명)를 사용 하 여 로그인 하 여 계정을 이동 해야 합니다. 계정을 이동 하는 단계를 마친 후에 멤버 자격 관리를 시작 하 고 혜택에 액세스 하 고 멤버 자격을 관리할 수 있도록 하 여 팀에 사용자 역할 및 사용 권한을 할당할 수 있습니다. 현재 역량, 혜택, 위치 정보, 은행/세금 정보 인센티브 및 Partner University 액세스를 포함 하는 MCP 연결에 대 한 Microsoft 자동으로 마이그레이션됩니다.

현재 역량, 혜택, 위치 정보, 은행/세금 정보 인센티브 및 Partner University 액세스를 포함 하는 MCP 연결에 대 한 Microsoft 자동으로 마이그레이션됩니다.

**갱신 정책 어떻게 변경 됩니까?**

 파트너 센터에서는 갱신 창-다음 30 일에 연주기 에서입니다.

**파트너 센터로 이동 후 우리의 역량을 변경 되지 않은 남아 있습니까?**

예, compentencies 파트너 센터로 이동 하 여 받지 않습니다. 불일치를 발견 하는 경우에 문의 [지원](https://partner.microsoft.com/support)합니다.


 **이동 후에 클라우드 혜택, 기술 지원, Visual Studio 소프트웨어 혜택 등 혜택 변경 됩니까?**

 적격 혜택 변경 되지 않습니다. 불일치를 발견 하는 경우에 문의 [지원](https://partner.microsoft.com/support)합니다.

 **Visual Studio 혜택 할당 된이 Microsoft 계정을 적용 됩니다.**

 예 Msa에 할당 된 Wisual Studio 혜택 적용 되어 유지 됩니다. 파트너 센터에서 새로 만든 후도 유지 됩니다. 그러나 제거 하는 경우 MSA 할당 한 번 마이그레이션된 파트너 센터에서, 파트너 센터에 다시 추가할 수 없습니다.

MPN 관리자만 계정 회사 Azure AD 테 넌 트에서 작동 하도록 Visual Studio 혜택 할당을 추가할 수 있습니다.

여러 AAD 테 넌 트의 경우 모든 테 넌 트에 동일한 파트너 센터 계정에 연결 된 전역 관리자는 Visual Studio 혜택 및 Azure 사용량 기반 할당에 이러한 모든 테이 넌 트에서 사용자를 추가할 수 있습니다. 

 **우리의 MCP 연결과 Partner University 액세스는 어떻게 관리 해야 합니까?**

 PMC에서 이동 하는 MCP 연결 변경이 있습니다. 그러나 모든 새 파트너 센터로 이동한 후의 새 직원 파트너 센터에 연결 해야 합니다. 기존 사용자에 대 한 모든 Partner University 권한을 상태로 유지 되지만 새 직원이 한 명도로 이동 해야 [교육 센터](https://partner.microsoft.com/training) Partner University에 대 한 액세스 권한을 얻으려고 하는 방법에 대 한 정보에 대 한 합니다.

 **파트너 센터에서 고객 참조 사용 됩니까?**

 아니요, 파트너 센터에서 역량 요구 사항에 대 한 고객 참조 필요는 없습니다.

 **Partner of Record 연결 파트너 센터로 이동 됩니다?**

 예, Partner of Record를 변하지가 않습니다. 자세히 알아보세요 [고객에 게 파트너 ID 연결](https://docs.microsoft.com/azure/billing/billing-partner-admin-link-started)합니다.

**영향을 미치는 인센티브는 무엇입니까?**

인센티브에 영향은 없습니다. 또한 비즈니스 PMC에 대 한 여러 계정 및 파트너 센터로 이동 이러한 계정이 글로벌 계정으로 통합 됩니다, 경우 레코드 기반 인센티브의 파트너에 게 영향을 주지 않습니다.

**파트너 센터에서 인센티브 사용자 역할은 무엇입니까?** 

파트너 센터에서 인센티브 역할 위치 기반 있으며 인센티브 관리자 및 인센티브 사용자를 포함 합니다. 이러한 역할에서 수행할 수 있는 작업에 대 한 자세한 내용은 참조 하세요. [사용자 역할 및 사용 권한을 할당할](permissions-overview.md)합니다.

**전역 및 위치 수준의 인센티브를 관리할 수 있습니까?**

 예 모든 위치에 대 한 인센티브 관리자 인센티브 관리자를 할당할 수 있습니다. 있습니다 또는 각 위치에는 자체 인센티브 관리자 수

**조회에 대 한 개수 비즈니스 프로필에서는 만들 수 있나요?**

회사는 여러 비즈니스 프로필에 대 한 필요성으로 회사의 관심사를 완벽 하 게 나타내고으로 만들 수 있습니다. 각 비즈니스 프로필에 5 개의 위치, 국가별로 한 위치까지 나열할 수 있습니다. 각 비즈니스 프로필의 각 해당 위치에 대 한 조회를 받을 수 있습니다.

**할당할 조회 되는 방식, 어떤 변경 사항을 예상할 수 있습니까? 예를 들어 다른 시장에서 한 시장에서 글로벌 회사 및 위치를 질문이 있으면 어떻게는 조회 수 할당 됩니까?**

조회는 고객을 정의 하는 검색 매개 변수를 기반으로 할당 됩니다. 이 한 위치 또는 여러 열을 원하는 위치를 지정 하는 고객 및 비즈니스 있는 다른 매개 변수를 충족 하는 조회는 해당 위치로 이동 해야 하는 경우 그렇게 합니다.
