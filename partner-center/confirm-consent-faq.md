**Microsoft는 고객에 게 Microsoft 클라우드 계약에서 수락한 있는지 확인 하는 파트너를 요구 하는 이유**

Microsoft 클라우드 계약의 올바른 실행 등 여러 가지 중요 한 항목이 상호 맞춤 함으로써 파트너, 고객 및 Microsoft 보호 보안, 개인 정보 및 데이터 보호에 제한 되지 않음. 확인 과정 투명도 및 맞춤 보장 하며 더 신속 하 게 규제 조회에 응답 해야 하는 고객을 지원 하기 위해 Microsoft 있습니다.

-   계약 승인 날짜
-   첫 번째 및 마지막 이름
-   이메일 주소
-   전화 번호 (선택 사항) 

**새로 만들기 | 우리가 기한에 맞지 않는 경우 어떻게 됩니까?**

파트너는 2018 년 8 월 7 일에는 기능을 사용할 수는 즉시 동의 확인을 시작할 수 있습니다. 파트너 센터 대시보드 사용자 2018 년 11 월 7부터 모든 새 거래에서 Microsoft 클라우드 계약 고객의 동의 확인 하려면 필요 합니다. 2019 년 3 월 22에 파트너 센터 API 사용자 동일한 작업을 수행 해야 합니다.

적용 당시 수 기존 구독에 실제 사용자를 기존 고객 또는 변경에 대 한 새 구매 넣을 수 없습니다 고객이 동의 확인 될 때까지 합니다. 새 고객에 대 한 거래 넣을 수 없습니다 받아들이는 확인 될 때까지 합니다. 자동 갱신은 받지이 요구 합니다.

**어떤 지역이 요구이 사항은 적용 되나요?**

**파트너는 고객의 Microsoft 클라우드 계약에 동의 확인 책임이?**

유효 2018 년 11 월 7 클라우드 서비스 공급자 (CSP) 간접 공급자와 CSP 직접 청구 파트너는 고객의 동의 확인 하는 일을 담당 모두와 이렇게 하려면 파트너 센터 API 또는 파트너 센터 대시보드에서 사용할 수 있습니다. 간접 공급자 먼저 간접 재판매인에서 필요한 정보를 수집 하 고 다음 Microsoft 파트너 센터 API 또는 파트너 센터 대시보드를 통해 사용할 수 있도록 해야 합니다. 계약상, 간접 공급자 CSP 재판매인의 최종 고객에 게 Microsoft 클라우드 계약에 동의 했는지 확인 하기 위해 Microsoft와 계약 바인딩이 로부터 것입니다. 

**수락 하 고 Microsoft 클라우드 계약 고객의 확인을 요청 하는 고객 시나리오는 무엇입니까?**

1.  새 고객을 새 구매 완료
2.  기존 고객을 새 구매 완료
3.  기존 구독에 실제 사용자 수를 업데이트 하는 기존 고객

**새 구입에서 별도 기존 고객에 대 한 동의 확인 하거나 추가 실제 사용자 수 있나요?**

Microsoft 클라우드 계약에서 수락한, 파트너 센터에서 고객 계정 섹션에 남아이 정보를 고객 적용 않는 시간 기간에 대 한 동의 다시 확인 하는 데 필요한 것을 확인 한 번 계약 조건입니다. 하면 새 주문을 배치 하기 전에 동의 다시 확인 해야 합니다.

Microsoft 클라우드 계약 갱신 하기 전에 업데이트 되는 경우, 파트너는 고객이 Microsoft 클라우드 계약 고객 승인을 즉시 받기 다시 필요는 없습니다 되지만 동의 Microsoft 클라우드 계약 사전에 다시 확인 해야 합니다. 갱신 합니다. 그러나 Microsoft 클라우드 계약의 새로운 기능과 향상 된 약관을 활용 하려면 갱신 하기 전에 새 조건에 동의 하는 고객이 수도 있습니다.

**새로 만들기 | 익숙한 없음 Microsoft 내 최종 고객의 직접 연락처 정보를 제공 합니다. Microsoft에서는이 정보를 사용 하는 방법**

파트너는 고객 테 넌 트를 만들 때에 고객에 대 한 기본 연락처 정보를 제공 해야 합니다. 온라인 서비스 약관 당 "Microsoft는 개인 데이터 처리 고객 으로부터 문서화 된 지침에 대해서만."

## <a href="" id="technicalfaq"></a>기술

**어떻게 파트너가 고객의 동의 확인 하 고 Microsoft에 제공? 이 프로세스를 지원 하기 위해 사용할 수 있는 도구?**

2018 년 8 월 7부터 사용할 수 있는 두 가지 방법은 파트너 지원 하기 위해: 
1.  파트너는 고객의 동의 프로그래밍 방식으로 확인 하려면 파트너 센터 API를 사용할 수 있습니다. 
2.  파트너는 고객의 동의 확인 하려면 파트너 센터 대시보드에서 사용할 수 있습니다.

**새로 만들기 | 저는 파트너 센터 API를 사용 하는 CSP 파트너입니다. 어떤 API 변경 내용 고려해 야 할 필요 한가요?** 

이 경우에 맞게 API 변경 내용 다음과 같습니다.

-   [https://docs.microsoft.com/en-us/partner-center/develop/get-confirmation-of-customer-consent](https://docs.microsoft.com/en-us/partner-center/develop/get-confirmation-of-customer-consent) 

-   [https://docs.microsoft.com/en-us/partner-center/develop/get-agreement-metadata](https://docs.microsoft.com/en-us/partner-center/develop/get-agreement-metadata)

-   [https://docs.microsoft.com/en-us/partner-center/develop/confirm-customer-consent](https://docs.microsoft.com/en-us/partner-center/develop/confirm-customer-consent)

**파트너 센터 API를 구현 하는 데 어려움이 데 어떻게 될까요?**

파트너 센터 대시보드를 통해 수동으로 수락을 확인할 수 있습니다.

**새로 만들기 | 파트너 센터 대시보드에서 또는 파트너 센터 API를 사용 하는 대신, 일괄 업로드할 수 고객 확인 정보를 사용 하 여 파일 대신?**

이 이번에 파트너 센터 대시보드에서 파트너 센터 API는 유일한 허용된 수단 수락 하 고 Microsoft 클라우드 계약 고객의 확인을 제공 합니다.

**새로 만들기 | 저는 제 3 자 플랫폼 또는 "미들웨어," 파트너 센터를 사용 하 여 회사의 시스템에 연결 하는 기능을 수행 하는 사용 하 여 파트너입니다. 어떻게 해야 하나요?**

파트너 센터 API를 통합 하는 플랫폼을 사용 하 여 파트너는 고객 확인 프로세스를 자동화 하는 파트너 센터 API 기능을 통합 하기 위해 자신의 계획 및 타임 라인에 대 한 공급자를 사용 하 여 직접 이야기 하 것이 좋습니다.

**새로 만들기 | 고객 전체 이름, 메일 주소 및 Microsoft 클라우드 계약 승인 날짜를 가져오는 데 API 흐름 예제를 제공할 수 있나요?**

자세한 내용과 예제 파트너 센터 API 설명서를 검토 하려면 파트너를 사용 하는 것이 좋습니다.

**새로 만들기 | 최신 Microsoft 클라우드 계약서를 고객에 게 표시 하 고 내 API 프로세스의 일부로 확인 공식에서 사용 하 여 다운로드 어떻게 해야 하나요?**

현재 파트너 센터 API는 매우 기본적인 프로그래밍 방식으로 Microsoft 클라우드 계약의 복사본을 검색 하는 파트너는 제공 하지 않으며 변경 사항을 버전 확인의 일환으로 고객에 허용 되 고 Microsoft 클라우드 계약의 캡처. 

**새로 만들기 | 파트너 센터 API를 사용 하 여이 프로세스에 대 한 userID 매개 변수가 필요 합니다. Azure AD와 통합 하지 않는 제 3 자 플랫폼을 통해 주문 놓으면이 요구를 관리 하는 방법**

이 경우 userID 고객 동의 확인을 제공 하는 파트너 테 넌 트 직원을 식별 합니다. 직원에 게 파트너의 테 넌 트 해당 Azure AD 사용자 계정이 없는 경우 해당 용도 대 한 특정 사용자 계정 지정을 제공 하는 직원의 id를 별도로 캡처 프로세스를 만드는 것이 좋습니다 합니다 확인 합니다. 

**새로 만들기 | Microsoft는 고객에 게 직접 가져오는 대신 Microsoft 클라우드 계약 고객 동의 확인 하는 파트너를 요구 하는 이유**

고객은 Microsoft에서 직접 CSP 파트너 로부터 구입 합니다. 서비스를 사용 하는 것에 대 한 Microsoft는 Microsoft 제공 용어를 Microsoft 클라우드 계약에 설명 되어 있습니다. Microsoft 클라우드 재판매인 계약 CSP 파트너에 게 고객 들이 검토 하 고 주문 하기 전에 Microsoft 클라우드 계약에 동의 해야 합니다. 

**새로 만들기 | API와 파트너 센터 대시보드 웹 사용자 인터페이스 사용 하 고 Microsoft 클라우드 계약 고객의 동의 확인 합니다. 2018 년 11 월 7 일 후 고객의 동의 확인 하려면 내 옵션은 무엇입니까?**

두 옵션 모두 사용 가능 하 고 Microsoft 클라우드 계약 고객의 동의 확인 하기 위해 사용할 수 있습니다. 파트너 센터 대시보드를 사용 하 여 거래 2018 년 11 월 7 제공 해야 하는 확인을 해야 합니다. 파트너 센터 API를 사용 하 여 거래 2019 년 3 월 22 시작 제공 해야 하는 확인을 해야 합니다.
