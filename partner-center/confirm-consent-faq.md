## <a href="" id="policyfaq"></a>정책

**Microsoft 고객은 Microsoft Cloud 계약에 동의한 있는지 확인 하는 파트너를 요구 하는 이유는?** 

Microsoft Cloud 계약 고객의 사용 권한, 약관 및 Microsoft 클라우드 제품 및 서비스 주문에 대 한 부여 조건을 제공 하 여 파트너, 고객 및 Microsoft를 보호 합니다. 

Microsoft Cloud 계약의 올바른 실행 파트너, 고객 및 Microsoft를 비롯 한 여러 중요 한 항목에 상호 맞춤 함으로써 보호 보안, 개인 정보 및 데이터 보호에 제한 되지 않음. 확인 프로세스 투명도 및 맞춤을 확인 하 고 더 신속 하 게 규정 질문에 응답 해야 하는 고객을 지원 하기 위해 Microsoft를 허용 합니다.

**고객 정보 확인 프로세스를 완료 하는 데 필요한 기능**
계약에 동의한 사람과 관련해 다음 정보가 필요합니다.
-   계약 동의 날짜
-   성과 이름
-   메일 주소
-   전화 번호(선택 사항) 

**새 | 에서는 기한을 충족 하지 않는 경우 어떻게 되나요?**

2018년 8월 7일에 이 기능이 출시되는 즉시 파트너는 동의 여부 확인을 시작할 수 있습니다. 2018 년 11 월 7 일 시작 파트너 센터 대시보드 사용자는 새 트랜잭션에서 모든 Microsoft 클라우드 계약의 고객 동의 확인 해야 합니다. 2019 년 3 월 22 일에서 파트너 센터 API 사용자가 동일한 작업을 수행 해야 합니다.

적용 시 기존 고객 또는 변경 내용을 기존 구독에 대 한 개수를 사용자에 대 한 새 구매 허용 되지 않습니다 고객 승인 확인 되어야 합니다. 신규 고객의 경우, 동의 여부가 확인될 때까지 거래가 허용되지 않습니다. 자동 갱신은 이 요구 사항의 영향을 받지 않습니다.

**지역은이 요구 사항이 적용 되나요?**

Microsoft 클라우드 계약에 고객 승인의 확인에서 Microsoft 공용 클라우드의 모든 지역에 대 한 필요합니다. 고객의 동의 여부 확인은 국가 클라우드(Microsoft Cloud for US Government 및 Microsoft 클라우드 독일)에서는 지원되지 않습니다.

**파트너는 고객의 Microsoft 클라우드 계약에 동의 확인 하는 일을 담당?**

일부 터 2018 년 11 월 7 일, 클라우드 서비스 공급자 (CSP) 간접 공급자 및 CSP 직접 청구 파트너 고객 승인 확인 하는 일을 담당 모두 이렇게 하려면 파트너 센터 API 또는 파트너 센터 대시보드를 사용할 수 있습니다. 간접 공급자 먼저 해당 간접 재판매인 으로부터 필요한 정보를 수집 하 고 다음 Microsoft 파트너 센터 API 또는 파트너 센터 대시보드를 통해 사용할 수 있도록 해야 합니다. 계약상으로 약속을 간접 공급자는 바인딩이 해당 CSP 재판매인의 최종 고객을 Microsoft 클라우드 계약에 동의한는 수 있도록 Microsoft와 계약 하는 것입니다. 

**고객 시나리오는 Microsoft 클라우드 계약의 고객 승인의 확인이 필요?**

CSP 파트너 트랜잭션은 확인 하려는 경우 Microsoft 클라우드 계약의 승인 확인 해야 합니다.
1.  새 고객을 신규 구매를 완료합니다.
2.  신규 구매를 완료 하는 기존 고객
3.  기존 구독에 사용자 수를 업데이트 하는 기존 고객

**해당 새 구매에서 별도 기존 고객에 대 한 동의 확인 하거나 추가 사용자 수 있나요?**

예. Microsoft 클라우드 계약에서 설명 하는 모든 고객에 대 한 고객의 동의 확인할 수 있습니다.

**여러 CSP 파트너는 고객에 대 한 요구 사항은 무엇입니까?**

동의 여부를 파트너가 확인해야 합니다. 한 고객이 여러 CSP 파트너와 거래하고 있다면 각 파트너가 동의 여부를 확인해야 합니다.

**저는 CSP 간접 공급자를 사용 하는 CSP 간접 재판매인입니다. 내 역할 이란?**

계약상으로 약속을 최종 사용자가 Microsoft 클라우드 계약을 수락 했습니다 수 있도록 Microsoft와 계약 바인딩이 있는 간접 공급자는 것입니다. 그러나 간접 공급자는 고객의 동의 여부를 수집하기 위해 간접 재판매인과 별도의 계약을 맺을 수 있습니다. 이 경우에 간접 공급자는 먼저 간접 재판매인으로부터 필요한 고객 동의 정보를 수집한 다음, 파트너 센터 API 또는 파트너 센터 대시보드를 통해 Microsoft에 이를 제공해야 합니다. 간접 재판매인은 요청 시 간접 공급자에게 필요한 정보를 신속하게 제공하여 이러한 프로세스를 지원할 수 있습니다.

**저는 CSP 간접 공급자입니다. 내 CSP 간접 재판매인 준비 하는 데 협력 해야 합니까?**

간접 재판매인으로부터 필요한 고객 동의 정보를 수집해야 하는 간접 공급자는 이러한 요구 사항을 간접 재판매인에게 곧 바로 알려서 이러한 정보의 제공 및 수령을 조율하는 것이 좋습니다.

**어떻게 수 설명할 내 고객에 게 Microsoft 클라우드 계약에 대 한 해당 동의 확인 해야 하면?**

Microsoft Cloud 계약의 올바른 실행 비롯 한 여러 중요 한 항목에 상호 맞춤 함으로써 Microsoft, 파트너 및 고객을 보호 보안, 개인 정보 및 데이터 보호에 제한 되지 않음. 확인 프로세스 투명도 및 맞춤을 확인 하 고 더 신속 하 게 규정 질문에 응답 해야 하는 고객을 지원 하기 위해 Microsoft를 허용 합니다.

갱신 하기 전에 Microsoft 클라우드 규약을 업데이트 하는 경우에서 파트너는 Microsoft 클라우드 계약의 고객 승인을 즉시 다시 얻어야 하는 고객이 아니지만 Microsoft Cloud 계약 이전에 대 한 동의 다시 확인 해야 합니다. 갱신 합니다. 그러나 Microsoft 클라우드 규약에서 새로운 기능과 향상 된 사용 약관을 활용 하려면 갱신 하기 전에 새 조건에 동의 하는 고객이 수도 있습니다.

**새 | 적당 하지 Microsoft 내 최종 고객의 직접 연락처 정보를 제공 합니다. Microsoft는이 정보를 사용 하는 방법**

파트너는 고객 테 넌 트를 만드는 경우에 고객에 대 한 기본 연락처 정보를 제공 하는 일을 해야 합니다. Online Services 약관을 당 "Microsoft는 데이터를 처리할 개인 고객의 문서화 된 지침에 대해서만."

## <a href="" id="technicalfaq"></a>기술

**어떻게 파트너 고객 동의 확인 하 고 Microsoft에 제공? 어떤 도구는이 프로세스를 지원 하 게 인가요?**

2018 년 8 월 7 일부터 사용할 수 있는 두 가지가 있습니다 파트너를 지원 하기 위해: 
1.  파트너가 파트너 센터 API를 통해 프로그래밍 방식으로 고객 동의를 확인할 수 있습니다. 
2.  파트너는 파트너 센터 대시보드를 사용 하 여 고객 동의 확인 수 있습니다.

**새 | 파트너 센터 API를 사용 하는 CSP 파트너인 데 어떤 API 변경 사항을 고려해 야 하나요?** 

이 경우 계정에 대 한 API 변경 내용이 다음과 같습니다.

-   [https://docs.microsoft.com/en-us/partner-center/develop/get-confirmation-of-customer-consent](https://docs.microsoft.com/en-us/partner-center/develop/get-confirmation-of-customer-consent) 

-   [https://docs.microsoft.com/en-us/partner-center/develop/get-agreement-metadata](https://docs.microsoft.com/en-us/partner-center/develop/get-agreement-metadata)

-   [https://docs.microsoft.com/en-us/partner-center/develop/confirm-customer-consent](https://docs.microsoft.com/en-us/partner-center/develop/confirm-customer-consent)

**그렇다면 파트너 센터 API를 구현 하는 데 문제가 있음?**

파트너 센터 대시보드를 통해 수동으로 동의 여부를 확인할 수 있습니다.

**새 | 파트너 센터 대시보드 또는 파트너 센터 API를 사용 하는 대신, 대량으로 업로드할 수 고객 확인 정보를 사용 하 여 파일 대신?**

이때 파트너 센터 대시보드 파트너 센터 API와 Microsoft 클라우드 계약의 고객 승인의 확인을 제공 하는 허용 된 유일한 수단입니다.

**새 | 저는 타사 플랫폼 또는 "미들웨어" 파트너 센터를 사용 하 여 회사의 시스템을 연결 하는 기능을 수행 하는 사용 하 여 파트너입니다. 수행 하나요?**

고객 확인 프로세스를 자동화 하는 파트너 센터 API 기능을 통합 하는 데 해당 계획 및 타임 라인에 대 한 공급자를 사용 하 여 직접 이야기 하는 파트너 센터 API를 통합 하는 플랫폼을 사용 하 여 파트너를 사용 하는 것이 좋습니다.

**새 | 고객의 전체 이름, 전자 메일 주소 및 Microsoft Cloud 계약 승인의 날짜 가져오기에 대 한 API 흐름 예제를 제공할 수 있습니다?**

세부 정보 및 예제에 대 한 파트너 센터 API 설명서를 검토 하려면 파트너를 사용 하는 것이 좋습니다.

**새 | 고객에 게 표시 하 고 확인 레코드에 내 API 프로세스의 일부로 사용할 최신 Microsoft Cloud 계약 가져오기**

현재 파트너 센터 API는 매우 기본적 이며 프로그래밍 방식으로 Microsoft 클라우드 계약 복사본을 검색 하는 파트너에 대 한 제공 하지 않습니다도 고객 동의한 사용자 확인의 일환으로 Microsoft 클라우드 계약의 버전 캡처합니다. 

**새 | 파트너 센터 API를 사용 하 여이 프로세스에 대 한 사용자 Id 매개 변수가 필요 합니다. Azure AD와 통합 하지 않는 타사 플랫폼을 통해 주문이 접수 된 경우이 요구 사항은 관리 하는 방법**

이 경우 사용자 Id의 고객 동의 확인을 제공 하는 파트너 테 넌 트 아래에 있는 직원을 식별 합니다. 직원에 해당 Azure AD 사용자 계정 파트너의 테 넌 트에 없는 경우를 캡처하는 프로세스를 개별적으로 제공 하는 직원의 id를 만들고 해당 목적을 위해 특정 사용자 계정을 지정 하는 것이 좋습니다는 확인 합니다. 

**새 | Microsoft 파트너가 고객 으로부터 직접 가져오는 대신 Microsoft Cloud 계약에 대 한 고객 동의 확인을 요구 하는 이유는?**

고객은 CSP 파트너가 Microsoft에서 직접 구입 합니다. 서비스 사용에 대 한 Microsoft는 Microsoft 제공 용어를 Microsoft 클라우드 계약에 설명 되어 있습니다. Microsoft 클라우드 거래 업체 규약에 CSP 파트너가 고객에 게 검토 하 고 주문 하기 전에 Microsoft 클라우드 계약에 동의 하도록 해야 합니다. 

**새 | API 및 파트너 센터 대시보드 웹 사용자 인터페이스는 Microsoft Cloud 계약 한 고객 동의 확인 하는 데 사용 합니다. 2018 년 11 월 7 일 후 고객 승인 확인 옵션이 이란?**

옵션을 모두 제공 되며 Microsoft Cloud 계약 고객 승인 확인 적합 합니다. 파트너 센터 대시보드를 사용 하 여 트랜잭션을 시작 2018 년 11 월 7 일에 존재 하도록 확인을 해야 합니다. 파트너 센터 API를 사용 하 여 트랜잭션을 시작 하는 2019 년 3 월 22 일에 존재 하도록 확인을 해야 합니다.
