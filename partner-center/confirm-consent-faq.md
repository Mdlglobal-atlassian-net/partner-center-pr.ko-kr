**고객의 Microsoft 클라우드 계약 (MCA)의 동의 확인 하는 새 프로세스 란?**

Microsoft는 고객을 지원 하기 위해 노력 하 고 파트너 규격 하세요. Microsoft의 클라우드 솔루션 공급자 (CSP) 프로그램의 파트너는 검토 Microsoft 사용 약관 및 Microsoft 제품 및 고객 서비스의 사용 제한을 포함 하 여 조건에 대 한 책임이 있습니다. 규정 준수 요구 사항을 충족 하는 파트너를 더 잘 지원 하 고 투명도 확인 CSP 파트너가 고객에 게 MCA 약관을 수락한 있는지 확인 하는 새 프로세스를 사용 했습니다. 2018 년 8 월 7 일 기준 파트너 센터 대시보드에서 사용자 인터페이스에도이 정보를 제공 하기 위해 파트너는 파트너 센터 API에 대 한 새 필드를 도입 되었습니다. 처음에 것이 확인 하기 위해 선택적으로 필요한 수 있지만 2018 년 11 월 7 일을 시작 합니다. 

**발효이 요구 사항은 이동 하는 경우**

파트너는 2018 년 8 월 7에 기능을 사용할 수는 즉시 동의 확인을 시작할 수 있습니다. 파트너 센터 대시보드에서 사용자 2018 년 11 월 7부터 모든 새 거래에 MCA의 고객이 동의 확인 하려면 필요 합니다. 2019 년 3 월 22에 파트너 센터 API 사용자 동일한 작업을 수행 해야 합니다. 기한 API 사용자에 대 한 추가 수 있도록 확장 되었습니다 기술 구현에 대 한 시간입니다.

**새로 만들기 | Microsoft는 파트너가 제공한 고객 정보의 정확성을 확인?**

Microsoft 제공 고객 전자 메일 주소 올바른 형식 인지 확인 합니다. 파트너의 책임을 정확 하 게 발견 되는 모든 정보를 수정 하 고 정확한 정보를 제공 하는 것입니다. 

**새 | 어떻게 파트너를 증명할 수의 MCA, 고객의 동의 문서화, 서명 되었는지 여부 디지털화, 또는 종이에?**

파트너는 고객의 MCA 약관을 수락한 있는지 확인 하 고 자체 프로세스 및 레코드를 관리 해야 합니다. 파트너 센터 대시보드에서 또는 파트너 센터 API 프로세스를 사용 하 여 파트너는 고객의 MCA 약관을 수락한 있는지 확인 합니다.

**새 | 어떻게 Microsoft 검사가 확인 프로세스는 고객에 대 한 전체?**

11 월 7 2018 년 때 CSP 파트너는 파트너 센터 대시보드를 사용 하 여 새로운 또는 기존 고객에 대 한 트랜잭션을 시작, Microsoft의 시스템은 자동으로 확인 고객 확인 정보는 파트너 센터에 존재 하는 경우 대시보드입니다. 나타나지 않습니다 트랜잭션을 완료 해야 할 수 없습니다. 

3 월 22 2019 년 때 CSP 파트너는 파트너 센터 API를 사용 하 여 새로운 또는 기존 고객에 대 한 트랜잭션을 시작, 시작 Microsoft의 시스템은 자동으로 고객 확인 정보 인지 확인 존재 합니다. 존재 인 경우 거래 완료 해야 할 수 없습니다.

알림 프로세스가 있습니다.

## <a href="" id="policyfaq"></a>정책

**Microsoft는 고객의 MCA 수락한 있는지 확인 하는 파트너를 요구 하는 이유** 

Microsoft 클라우드 계약 (MCA)은 고객의 사용 권한, 약관 및 Microsoft 클라우드 제품 및 서비스 주문에 부여 하는 조건을 제공 하 여 파트너, 고객 및 Microsoft를 보호 합니다. 

MCA의 올바른 실행에 포함 하 여 여러 가지 중요 한 항목이 상호 맞춤 함으로써 파트너, 고객 및 Microsoft 보호 보안, 개인 정보 및 데이터 보호에 제한 되지 않음. 확인 과정 투명도 및 맞춤 보장 하며 더 신속 하 게 규제 조회에 응답 해야 하는 고객을 지원 하기 위해 Microsoft 있습니다.

**어떤 고객 정보를 확인 하는 프로세스를 완료 하는 데 필요한?**
계약 동의 사람에 대 한 다음 정보가 필요 합니다.
-   계약 승인 날짜
-   첫 번째 및 마지막 이름
-   이메일 주소
-   전화 번호 (선택 사항) 

**새 | 제출 날짜 형식이 없는 경우 어떻게 될까요?**

파트너는 2018 년 8 월 7에 기능을 사용할 수는 즉시 동의 확인을 시작할 수 있습니다. 파트너 센터 대시보드에서 사용자 2018 년 11 월 7부터 모든 새 거래에 MCA의 고객이 동의 확인 하려면 필요 합니다. 2019 년 3 월 22에 파트너 센터 API 사용자 동일한 작업을 수행 해야 합니다.

적용 당시 수 기존 구독에 실제 사용자를 기존 고객 또는 변경에 대 한 새 구매 넣을 수 없습니다 고객이 동의 확인 될 때까지 합니다. 새 고객에 대 한 거래는 허용 하지 않는 받아들이는 확인 될 때까지 합니다. 자동 갱신은 받지이 요구 합니다.

**어떤 지역이 요구이 사항은 적용 되나요?**

고객 계약의 MCA에 확인 Microsoft 공용 클라우드 아래에 있는 모든 지역에 대 한 필요 합니다. 고객의 동의 확인 국가별 (Microsoft 클라우드 독일 및 미국 정부용 Microsoft 클라우드 포함)를 사용 하 여 지원 되지 않습니다.

**파트너는 고객의 MCA는 동의 확인 할 책임이?**

유효 2018 년 11 월 7 일 클라우드 서비스 공급자 (CSP) 간접 공급자와 CSP 직접 청구 파트너는 고객의 동의 확인 하는 일을 담당 모두와 이렇게 하려면 파트너 센터 API 또는 파트너 센터 대시보드에서 사용할 수 있습니다. 간접 공급자 먼저 간접 재판매인에서 필요한 정보를 수집 하 고 다음 Microsoft 파트너 센터 API 또는 파트너 센터 대시보드를 통해 사용할 수 있도록 해야 합니다. 계약상, 간접 공급자 CSP 재판매인의 최종 고객의 MCA를 수락한 있는지 확인 하기 위해 Microsoft와 계약 바인딩이 로부터 것입니다. 

**수락 하 고 MCA 고객의 확인을 요청 하는 고객 시나리오는 무엇입니까?**

CSP 파트너 다음 거래를 확인 하려면 Microsoft 클라우드 계약의 동의 확인 해야 합니다.
1.  새 고객을 새 구매 완료
2.  기존 고객을 새 구매 완료
3.  기존 구독에 실제 사용자 수를 업데이트 하는 기존 고객

**새 구입에서 별도 기존 고객에 대 한 동의 확인 하거나 추가 실제 사용자 수 있나요?**

그렇습니다. 모든 고객의 MCA 덮인 것에 대 한 고객 동의 확인할 수 있습니다.

**여러 CSP 파트너는 고객에 대 한 요구 사항은 무엇입니까?**

동의 확인 파트너가 확인 됩니다. CSP는 여러 파트너와 고객 transacts, 하는 경우 각 파트너 동의 확인 해야 합니다.

**저는 CSP 간접 공급자와 협력 CSP 간접 재판매인입니다. 내 책임 란 무엇 인가요?**

계약상, 간접 공급자에 최종 고객의 MCA 수락한 있는지 확인 하기 위해 Microsoft와 계약 바인딩이 로부터 것입니다. 그러나 간접 공급자는 고객의 동의 수집 하려면 간접 재판매인과 별도 피어로 보유할 수 있습니다. 이 경우 간접 공급자 먼저 간접 재판매인에서 필요한 고객 동의 정보를 수집 하 고 다음 Microsoft 파트너 센터 API 또는 파트너 센터 대시보드를 통해 사용할 수 있게 할 수 있습니다. 고객이 요청 시 간접 공급자를 신속 하 게 필요한 정보를 제공 하 여이 프로세스를 지원할 수 있습니다.

**저는 CSP 간접 공급자입니다. 준비 CSP 간접 재판매인 내에 맞게 해야 합니까?**

배달 및 수신이 정보를 조정 하는 즉시이 요구 사항은 간접 재판매인을 통신할 간접 재판매인에서 필요한 고객 동의 정보를 수집 해야 하는 간접 공급자를 사용 하는 것이 좋습니다.

**수 있나요에 어떻게 설명 고객은 해당 수락 하 고 MCA 되었는지 확인 해야?**

MCA의 올바른 실행에 포함 하 여 여러 가지 중요 한 항목이 상호 맞춤 함으로써 Microsoft, 파트너와 고객을 보호 보안, 개인 정보 및 데이터 보호에 제한 되지 않음. 확인 과정 투명도 및 맞춤 보장 하며 더 신속 하 게 규제 조회에 응답 해야 하는 고객을 지원 하기 위해 Microsoft 있습니다.

**새로 만들기 | 고객이 단일 CSP 간접 공급자와 함께 작동 하는 경우 여러 CSP 간접 재판매인으로 거래 완료 되는 각 CSP 간접 재판매인 제공할 필요가 CSP 간접 공급자 확인 정보를 사용 하 여?**

CSP 간접 공급자는 Microsoft로의 MCA 고객이 동의 확인을 제공 해야 합니다. 각 고객 계정에 대해 하나의 확인 파트너 센터 대시보드에서 한 번에 기록할 수 있습니다. 

CSP 간접 공급자는 고객 연락 담당자는 MCA 동의 담당 CSP 간접 재판매인으로 거래 하는 번호와 상관 없이 따라서 완료가 확인 해야 합니다.

**새 | 고객 조직 내 MCA의 동의 확인 하려면 권한이 있는 사용자 수 있어야 하는 작업**

것은 파트너의 책임 고객 조직 내 사용자에 게는 MCA의 고객 승인을 받아야 합니다. 것은 동의 제공 하기 위해 권한이 있는 사용자를 확인 하는 고객의 책임입니다. 

**새 | MCA는 버전의 고객이 동의 확인 하는 일을 담당 하는 파트너는? 파트너는 MCA 버전 고객에서 수락한 나타내는 해야 합니까?**

파트너는 새 고객과 CSP 거래를 만들 때 최신 MCA를 사용 하 여 책임이 있습니다. 파트너 최신 MCA 갱신, 변경 또는 기존 고객을 사용 하 여 새 트랜잭션을 완료 하는 경우의 동의 확인 해야 합니다. 

MCA는 갱신 하기 전에 업데이트 되는 경우, 파트너는 MCA 고객 승인을 즉시 받기 다시 고객에 게 필요는 없습니다 되지만 갱신 하기 전에 MCA 동의 다시 확인 해야 합니다. 그러나 활용 하기 위해 신규 및 개선 용어는 MCA에서 갱신 하기 전에 새 조건에 동의 하는 고객이 수도 있습니다.

**새 | 저는 CSP 간접 재판매인입니다. 내 회사 레코드에는 MCA 수락 하는 고객을 기록 I 경우 파트너 센터 대시보드에서 또는 파트너 센터 API를 통해이 정보를 입력 해야 하나요?**

계약상, 간접 공급자에 최종 고객의 MCA 수락한 있는지 확인 하기 위해 Microsoft와 계약 바인딩이 로부터 것입니다. 그러나 간접 공급자는 고객의 동의 수집 하려면 간접 재판매인과 별도 피어로 보유할 수 있습니다. 이 경우 간접 공급자 먼저 간접 재판매인에서 필요한 고객 동의 정보를 수집 하 고 다음 Microsoft 파트너 센터 API 또는 파트너 센터 대시보드를 통해 사용할 수 있게 할 수 있습니다.

**새 | 그렇다면 내 고객의 MCA 약관에 일치 하지 않는?**

Microsoft의 CSP 프로그램 파트너는 고객의 주문 하기 전에 MCA의 동의 대 한 책임이 있습니다. 새 요구 사항이 아닙니다. 어떤 이유로 든 MCA, 고객이 더 이상 일치 하는 경우 다음 파트너 취소 해야이 고객과 관련 된 구독 합니다.

**새로 만들기 | 재판매인 고객을 대신 하 여 동의 제공할 수 있습니다.**

아니요.

**고객이 동의 제공 했습니다 있는지를 확인 한 후 나중에 고객에 대 한 확인 다시 해야 하나요?**

한 번 되었다고 확인는 고객의 MCA 수락한,이 정보는 파트너 센터에서 고객 계정 섹션에 남아 및 고객 계약 조건에 적용 하는 기간에 대 한 동의 다시 확인 필요는 없습니다. 새 주문을 배치 하기 전에 동의 다시 확인 고만 해야 합니다.

MCA 갱신 하기 전에 업데이트 되는 경우, 파트너는 MCA 고객 승인을 즉시 받기 다시 고객에 게 필요는 없습니다 되지만 갱신 하기 전에 MCA 동의 다시 확인 해야 합니다. MCA는 새롭고 향상 된 약관을 활용 하는 갱신 하기 전에 새 조건에 동의 하는 고객에 게 수도 있습니다.

**새 | 본인에 게 편안한 없음 Microsoft 내 최종 고객 직접 연락처 정보를 제공 합니다. Microsoft에서는이 정보를 사용 하는 방법**

파트너는 고객 테 넌 트를 만들 때에 고객에 대 한 기본 연락처 정보를 제공 해야 합니다. 온라인 서비스 약관을 당 "Microsoft는 개인 데이터 처리 고객 으로부터 문서화 된 지침에 대해서만."

## <a href="" id="technicalfaq"></a>기술

**어떻게 파트너가 고객의 동의 확인 하 고 Microsoft에 제공? 이 프로세스를 지원 하기 위해 사용할 수 있는 도구?**

2018 년 8 월 7부터 사용할 수 있는 방법은 두 파트너 지원 하기 위해: 
1.  파트너는 프로그래밍 방식으로 고객의 동의 확인 하려면 파트너 센터 API를 사용할 수 있습니다. 
2.  파트너는 고객의 동의 확인 하려면 파트너 센터 대시보드에서 사용할 수 있습니다.

**새로 만들기 | 저는 파트너 센터 API를 사용 하는 CSP 파트너입니다. 어떤 API 변경 내용 고려해 야 할 필요 한가요?** 

이 경우 API 변경 내용을 고려 다음과 같습니다.

-   [https://docs.microsoft.com/en-us/partner-center/develop/get-confirmation-of-customer-consent](https://docs.microsoft.com/en-us/partner-center/develop/get-confirmation-of-customer-consent) 

-   [https://docs.microsoft.com/en-us/partner-center/develop/get-agreement-metadata](https://docs.microsoft.com/en-us/partner-center/develop/get-agreement-metadata)

-   [https://docs.microsoft.com/en-us/partner-center/develop/confirm-customer-consent](https://docs.microsoft.com/en-us/partner-center/develop/confirm-customer-consent)

**어떻게 파트너 센터 API를 구현 하는 데 어려움을 겪고 있나요?**

파트너 센터 대시보드를 통해 수동으로 수락을 확인할 수 있습니다.

**새 | 파트너 센터 대시보드에서 또는 파트너 센터 API를 사용 하는 대신, 일괄 업로드할 수 고객 확인 정보를 사용 하 여 파일 대신?**

이 이번에 파트너 센터 대시보드에서 파트너 센터 API는 유일한 허용 된 수단 수락 하 고 MCA 고객의 확인을 제공 합니다.

**새 | 저는 제 3 자 플랫폼 또는 "미들웨어," 파트너 센터를 사용 하 여 회사의 시스템에 연결 하는 기능을 수행 하는 사용 하 여 파트너입니다. 어떻게 해야 하나요?**

파트너 센터 API를 통합 하는 플랫폼을 사용 하 여 파트너는 고객 확인 프로세스를 자동화 하는 파트너 센터 API 기능을 통합 하기 위해 자신의 계획 및 타임 라인에 대 한 공급자를 사용 하 여 직접 이야기 하 것이 좋습니다.

**새 | 고객 전체 이름, 메일 주소 및 MCA 수용 날짜를 가져오는 데 API 흐름 예제를 제공할 수 있나요?**

자세한 내용과 예제 파트너 센터 API 설명서를 검토 하려면 파트너를 사용 하는 것이 좋습니다.

**새 | 고객에 게 표시 하 고 내 API 프로세스의 일부로 확인 레코드에 사용할 최신 MCA 다운로드 어떻게 해야 하나요?**

현재 파트너 센터 API는 매우 기본적인 MCA의 복사본을 프로그래밍 방식으로 검색할 파트너는 제공 하지 않으며 버전 확인의 일환으로 고객을 수락한 MCA의 캡처. 

**새 | 파트너 센터 API를 사용 하 여이 프로세스에 대 한 userID 매개 변수가 필요 합니다. Azure AD와 통합 하지 않는 제 3 자 플랫폼을 통해 주문 놓으면이 요구를 관리 하는 방법**

이 경우 userID 직원의 고객 동의 확인을 제공 하는 파트너 테 넌 트를 식별 합니다. 직원이 파트너의 테 넌 트를 해당 Azure AD 사용자 계정이 없는 경우 별도로 제공 하는 직원의 id를 캡처하는 프로세스를 만들고 해당 용도 대 한 특정 사용자 계정 지정 하는 것이 좋습니다 합니다 확인 합니다. 

**새로 만들기 | Microsoft는 고객에 게 직접 획득 하는 대신 MCA 고객이 동의 확인 하려면 파트너를 요구 하는 이유**

고객은 Microsoft에서 직접 CSP 파트너는에서 구입 합니다. 서비스를 사용 하는 것에 대 한 Microsoft는 Microsoft 제공 용어를 MCA에 기록 됩니다. MCRA는 CSP 파트너가 고객 검토 하 고 주문 하기 전에 MCA 동의 필요 합니다. 

**새로 만들기 | API와 파트너 센터 대시보드 웹 사용자 인터페이스 사용 하 고 MCA 고객의 동의 확인 합니다. 2018 년 11 월 7 일 후 고객의 동의 확인 하려면 내 옵션은 무엇입니까?**

두 옵션 모두 사용 가능 하 고 MCA 고객이 동의 확인 하기 위해 사용할 수 있습니다. 파트너 센터 대시보드를 사용 하 여 거래 2018 년 11 월 7 나타날 수를 확인을 해야 합니다. 파트너 센터 API를 사용 하 여 거래 2019 년 3 월 22 시작 나타날 수를 확인을 해야 합니다.
