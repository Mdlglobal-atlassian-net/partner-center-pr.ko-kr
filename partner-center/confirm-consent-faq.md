## <a href="" id="processfaq"></a>프로세스

**Microsoft 클라우드 계약에 대 한 고객의 동의를 확인 하는 새로운 프로세스는 무엇 인가요?**

Microsoft는 고객과 파트너가 규정을 준수하도록 돕고 있습니다. Microsoft CSP (클라우드 솔루션 공급자) 프로그램의 파트너는 고객과의 Microsoft 제품 및 서비스에 대 한 적절 한 사용을 비롯 하 여 Microsoft의 사용 약관을 검토 해야 합니다. 회의 준수 요구 사항에 대 한 파트너의 도움을 높이고 투명성을 보장 하기 위해 CSP 파트너가 필요한 새 프로세스를 사용 하도록 설정 하 여 고객이 Microsoft 클라우드 계약 약관에 동의 했는지 확인 합니다. 2018 년 8 월 7 일부 터 파트너 센터 대시보드 사용자 인터페이스에 새 필드를 도입 했으며 파트너에 게이 정보를 제공 하기 위한 파트너 센터 API도 도입 했습니다. 처음에는이 확인을 제공 하는 옵션을 선택 해야 하지만, 2018 년 11 월 7 일부 터는 필수 항목입니다. 

**이 요구 사항이 적용 되는 시기는 언제 입니까?**

2018년 8월 7일에 이 기능이 출시되는 즉시 파트너는 동의 여부 확인을 시작할 수 있습니다. 2018 년 11 월 7 일부 터 파트너 센터 대시보드 사용자는 모든 새 트랜잭션에서 Microsoft 클라우드 계약에 대 한 고객의 동의를 확인 해야 합니다. 2019 년 3 월 22 일에 파트너 센터 API 사용자는 동일한 작업을 수행 해야 합니다. API 사용자의 최종 기한이 기술 구현에 대 한 추가 시간을 허용 하도록 확장 되었습니다.

**새로 만들기 | Microsoft에서 파트너가 제공 하는 고객 정보의 정확성을 확인 하나요?**

Microsoft에서는 제공 된 고객 전자 메일 주소가 올바른 형식 인지 확인 합니다. 정확한 정보를 제공 하 고 정확 하지 않은 것으로 확인 된 정보를 수정 하는 것은 파트너의 책임입니다. 

**새로 만들기 | 파트너가 고객에 게 Microsoft 클라우드 계약에 대 한 동의를 입증 하는 방법 (서명, 문서화, 디지털 또는 페이퍼)은 무엇 인가요?**

파트너는 고객이 Microsoft 클라우드 계약의 약관에 동의 하 고 자신의 프로세스와 레코드를 관리 하는 일을 담당 합니다. 파트너는 파트너 센터 대시보드 또는 파트너 센터 API 프로세스를 사용 하 여 고객이 Microsoft 클라우드 계약의 약관에 동의 했는지 확인 합니다.

**새로 만들기 | Microsoft에서 고객에 대 한 확인 프로세스가 완료 되었는지 어떻게 확인 하나요?**

2018 년 11 월 7 일부 터 CSP 파트너가 신규 또는 기존 고객에 대 한 파트너 센터 대시보드를 사용 하 여 트랜잭션을 시작 하면 Microsoft 시스템은 고객 확인 정보가 파트너 센터에 있는지 자동으로 확인 합니다. 대시보드와. 해당 정보가 없는 경우 거래를 완료할 수 없습니다. 

2019 년 3 월 22 일부 터 CSP 파트너가 파트너 센터 API를 사용 하 여 신규 또는 기존 고객에 대 한 트랜잭션을 시작 하면 Microsoft 시스템은 고객 확인 정보가 있는지 자동으로 확인 합니다. 해당 정보가 없는 경우 거래를 완료할 수 없습니다. 

**새로 만들기 | 이메일 주소가 확인 프로세스에서 제공 되는 고객에 게 알림이 전송 되나요?**

알림 프로세스가 없습니다.

## <a href="" id="policyfaq"></a>Policy

**Microsoft에서 고객이 Microsoft 클라우드 계약에 동의 했는지 확인 하는 데 파트너가 필요한 이유는 무엇 인가요?** 

Microsoft 클라우드 계약은 Microsoft 클라우드 제품과 서비스 주문에 대해 부여되는 고객의 사용 권한, 사용 약관과 조건을 제공함으로써 파트너, 고객 및 Microsoft를 보호합니다. 

Microsoft 클라우드 규약을 적절 하 게 실행 하면 보안, 개인 정보 보호 및 데이터 보호를 비롯 한 여러 중요 항목에 대해 상호 정렬을 수행 하 여 파트너, 고객 및 Microsoft를 보호할 수 있습니다. 확인 프로세스를 통해 투명도와 맞춤 기능을 보장 하 고 Microsoft는 규정 질문에 즉시 대응 해야 하는 고객을 보다 잘 지원할 수 있습니다.

**확인 프로세스를 완료 하는 데 필요한 고객 정보는 무엇 인가요?**
계약에 동의한 사람과 관련해 다음 정보가 필요합니다.
-   계약 동의 날짜
-   성과 이름
-   전자 메일 주소
-   전화 번호(선택 사항) 

**새로 만들기 | 기한이 충족 되지 않으면 어떻게 되나요?**

2018년 8월 7일에 이 기능이 출시되는 즉시 파트너는 동의 여부 확인을 시작할 수 있습니다. 2018 년 11 월 7 일부 터 파트너 센터 대시보드 사용자는 모든 새 트랜잭션에서 Microsoft 클라우드 계약에 대 한 고객의 동의를 확인 해야 합니다. 2019 년 3 월 22 일에 파트너 센터 API 사용자는 동일한 작업을 수행 해야 합니다.

적용 시 고객 동의가 확인 될 때까지 기존 고객에 대 한 새로운 구매 또는 기존 구독에서의 사용자 수 변경 사항은 허용 되지 않습니다. 신규 고객의 경우, 동의 여부가 확인될 때까지 거래가 허용되지 않습니다. 자동 갱신은 이 요구 사항의 영향을 받지 않습니다.

**이 요구 사항이 적용 되는 지역은 어디 인가요?**

Microsoft 공용 클라우드의 모든 지역에는 Microsoft 클라우드 계약에 대 한 고객 승인 확인이 필요 합니다. 고객의 동의 여부 확인은 국가 클라우드(Microsoft Cloud for US Government 및 Microsoft 클라우드 독일)에서는 지원되지 않습니다.

**Microsoft 클라우드 계약에 대 한 고객의 동의를 확인 하는 파트너는 무엇입니까?**

11 월 7 일, 2018, CSP (클라우드 서비스 공급자) 간접 공급자 및 CSP 직접 청구 파트너는 모두 고객 승인 확인을 담당 하며 파트너 센터 API 또는 파트너 센터 대시보드를 사용 하 여이 작업을 수행할 수 있습니다. 간접 공급자는 먼저 간접 대리점에서 필요한 정보를 수집한 다음 파트너 센터 API 또는 파트너 센터 대시보드를 통해 Microsoft에서 사용할 수 있도록 해야 할 수 있습니다. 계약상, CSP 대리점의 최종 고객이 Microsoft 클라우드 계약에 동의할 수 있도록 Microsoft와의 바인딩 계약이 있는 간접 공급자입니다. 

**고객에 게 Microsoft 클라우드 계약에 대 한 승인 확인이 필요한 고객 시나리오는 무엇입니까?**

CSP 파트너가 다음과 같은 트랜잭션을 수행 하려는 경우 Microsoft 클라우드 규약을 승인 해야 합니다.
1.  새 고객이 새로운 구매를 완료 합니다.
2.  기존 고객이 새로운 구매를 완료 합니다.
3.  기존 구독에 대 한 기존 고객 업데이트 수

**새 구매 또는 사용자를 추가 하는 것과 별개의 기존 고객에 대 한 승인을 확인할 수 있나요?**

예 Microsoft 클라우드 계약에 적용 되는 모든 고객에 대해 고객 동의가 확인 될 수 있습니다.

**여러 CSP 파트너가 있는 고객을 위한 요구 사항은 무엇 인가요?**

동의 여부를 파트너가 확인해야 합니다. 한 고객이 여러 CSP 파트너와 거래하고 있다면 각 파트너가 동의 여부를 확인해야 합니다.

**CSP 간접 공급자와 함께 작동 하는 CSP 간접 재판매인입니다. 책임은 무엇 인가요?**

계약상 it는 최종 고객이 Microsoft 클라우드 규약을 수락 했는지 확인 하기 위해 Microsoft와의 바인딩 계약이 있는 간접 공급자입니다. 그러나 간접 공급자는 고객의 동의 여부를 수집하기 위해 간접 재판매인과 별도의 계약을 맺을 수 있습니다. 이 경우에 간접 공급자는 먼저 간접 재판매인으로부터 필요한 고객 동의 정보를 수집한 다음, 파트너 센터 API 또는 파트너 센터 대시보드를 통해 Microsoft에 이를 제공해야 합니다. 간접 재판매인은 요청 시 간접 공급자에게 필요한 정보를 신속하게 제공하여 이러한 프로세스를 지원할 수 있습니다.

**CSP 간접 공급자입니다. 준비를 위해 CSP 간접 재판매인과 조정 해야 하나요?**

간접 재판매인으로부터 필요한 고객 동의 정보를 수집해야 하는 간접 공급자는 이러한 요구 사항을 간접 재판매인에게 곧 바로 알려서 이러한 정보의 제공 및 수령을 조율하는 것이 좋습니다.

**고객에 게 Microsoft 클라우드 계약에 대 한 동의를 확인 해야 한다는 것을 어떻게 이해할 수 있나요?**

Microsoft 클라우드 규약을 적절 하 게 실행 하면 보안, 개인 정보 보호 및 데이터 보호를 비롯 한 여러 중요 항목에 대 한 상호 정렬을 통해 Microsoft, 파트너 및 고객을 보호 합니다. 확인 프로세스를 통해 투명도와 맞춤 기능을 보장 하 고 Microsoft는 규정 질문에 즉시 대응 해야 하는 고객을 보다 잘 지원할 수 있습니다.

**새로 만들기 | 고객이 단일 CSP 간접 공급자를 사용 하 여 작업 하지만 트랜잭션이 여러 CSP 간접 대리점으로 완료 되는 경우 각 CSP 간접 재판매인은 확인 정보를 사용 하 여 CSP 간접 공급자를 제공 해야 하나요?**

CSP 간접 공급자는 Microsoft에 대 한 Microsoft 클라우드 계약 고객 동의를 확인 하는 일을 담당 합니다. 각 고객 계정에 대해 한 번에 하나의 확인만 파트너 센터 대시보드에 기록할 수 있습니다. 

CSP 간접 공급자는이를 사용 하는 CSP 간접 대리점의 수에 관계 없이 Microsoft 클라우드 계약에 동의 하는 고객 담당자가 작업을 수행할 수 있는지 확인 해야 합니다.

**새로 만들기 | 고객 조직 내의 Microsoft 클라우드 계약에 대 한 동의를 확인 하는 권한이 있는 사용자는 누구 인가요?**

고객 조직 내부의 사용자 로부터 Microsoft 클라우드 계약에 대 한 고객의 동의를 얻는 것은 파트너의 책임입니다. 고객은 동의를 제공할 수 있는 권한이 있는 사용자를 결정 해야 합니다. 

**새로 만들기 | 파트너가 고객 승인 확인을 담당 하는 Microsoft 클라우드 규약의 버전은 무엇 인가요? 파트너가 고객에 게 동의한 Microsoft 클라우드 계약의 버전을 표시 해야 하나요?**

파트너는 새 고객에 게 CSP 트랜잭션을 만들 때 최신 Microsoft 클라우드 규약을 사용 해야 합니다. 파트너는 기존 고객에 대 한 새 트랜잭션을 갱신, 변경 또는 완료할 때 최신 Microsoft 클라우드 계약에 대 한 동의를 확인 해야 합니다. 

갱신 전에 Microsoft 클라우드 계약이 업데이트 된 경우에는 파트너가 고객에 게 즉시 Microsoft 클라우드 계약에 대 한 고객 동의를 다시 획득 하도록 요구 하지는 않지만 이전에는 Microsoft 클라우드 계약에 동의 해야 다시 확인 갱신 합니다. 그러나 Microsoft 클라우드 계약에서 새로운 기능 및 향상 된 용어를 활용 하기 위해 갱신 전에 고객이 새 약관에 동의 하도록 할 수도 있습니다.

**새로 만들기 | 저는 CSP 간접 대리점입니다. 회사 레코드의 Microsoft 클라우드 계약에 대 한 고객 동의를 기록 하는 경우 파트너 센터 대시보드 또는 파트너 센터 API를 통해이 정보를 제공 해야 하나요?**

계약상 it는 최종 고객이 Microsoft 클라우드 규약을 수락 했는지 확인 하기 위해 Microsoft와의 바인딩 계약이 있는 간접 공급자입니다. 그러나 간접 공급자는 고객의 동의 여부를 수집하기 위해 간접 재판매인과 별도의 계약을 맺을 수 있습니다. 이 경우에 간접 공급자는 먼저 간접 재판매인으로부터 필요한 고객 동의 정보를 수집한 다음, 파트너 센터 API 또는 파트너 센터 대시보드를 통해 Microsoft에 이를 제공해야 합니다.

**새로 만들기 | 고객이 Microsoft 클라우드 계약의 조건에 동의 하지 않으면 어떻게 되나요?**

Microsoft CSP 프로그램의 파트너는 주문 하기 전에 Microsoft 클라우드 규약을 고객에 게 동의한 책임이 있습니다. 이는 새로운 요구 사항이 아닙니다. 어떤 이유로 든 고객이 더 이상 Microsoft 클라우드 계약에 동의 하지 않으면 파트너가이 고객과 연결 된 구독을 취소 해야 합니다.

**새로 만들기 | 재판매인이 고객을 대신 하 여 동의를 제공할 수 있나요?**

아니요.

**고객이 승인을 제공 하는 것을 확인 한 후 나중에 고객에 게 다시 확인 해야 하나요?**

고객이 Microsoft 클라우드 계약에 동의 하는 것을 확인 한 후에는이 정보가 파트너 센터의 고객 계정 섹션에 남아 있으며, 고객에 게 적용 되는 기간에 대 한 승인을 다시 확인 필요가 없습니다. 규약 조건입니다. 새 주문을 배치 하기 전에 다시 확인만 허용 해야 합니다.

갱신 전에 Microsoft 클라우드 계약이 업데이트 된 경우에는 파트너가 고객에 게 즉시 Microsoft 클라우드 계약에 대 한 고객 동의를 다시 획득 하도록 요구 하지는 않지만 이전에는 Microsoft 클라우드 계약에 동의 해야 다시 확인 갱신 합니다. 그러나 Microsoft 클라우드 계약에서 새로운 기능 및 향상 된 용어를 활용 하기 위해 갱신 전에 고객이 새 약관에 동의 하도록 할 수도 있습니다.

**새로 만들기 | Microsoft의 최종 고객에 게 직접 연락 정보를 제공 하는 것은 불편 하지 않습니다. Microsoft는이 정보를 어떻게 사용 하나요?**

파트너는 고객 테 넌 트를 만들 때 고객에 대 한 기본 연락처 정보를 제공 해야 합니다. Microsoft는 온라인 서비스 약관에 따라 고객의 문서화 된 지침 에서만 개인 데이터를 처리 합니다. "

## <a href="" id="technicalfaq"></a>기술

**파트너는 고객 승인을 확인 하 고 Microsoft에 제공 하는 방법 이 프로세스를 지원 하기 위해 사용할 수 있는 도구는 무엇입니까?**

파트너를 지원 하기 위해 2018 년 8 월 7 일에 사용할 수 있는 두 가지 방법이 있습니다. 
1.  파트너가 파트너 센터 API를 통해 프로그래밍 방식으로 고객 동의를 확인할 수 있습니다. 
2.  파트너는 파트너 센터 대시보드를 사용 하 여 고객 승인을 확인할 수 있습니다.

**새로 만들기 | 파트너 센터 API를 사용 하는 CSP 파트너입니다. 고려해 야 할 API 변경 사항은 무엇 인가요?** 

이 경우의 계정에 대 한 API 변경 내용은 다음과 같습니다.

-   [https://docs.microsoft.com/partner-center/develop/get-confirmation-of-customer-consent](https://docs.microsoft.com/partner-center/develop/get-confirmation-of-customer-consent) 

-   [https://docs.microsoft.com/partner-center/develop/get-agreement-metadata](https://docs.microsoft.com/partner-center/develop/get-agreement-metadata)

-   [https://docs.microsoft.com/partner-center/develop/confirm-customer-consent](https://docs.microsoft.com/partner-center/develop/confirm-customer-consent)

**파트너 센터 API를 구현 하는 데 어려움이 있는 경우 어떻게 하나요?**

파트너 센터 대시보드를 통해 수동으로 동의 여부를 확인할 수 있습니다.

**새로 만들기 | 파트너 센터 대시보드 또는 파트너 센터 API를 사용 하는 대신, 고객 확인 정보를 사용 하 여 파일을 대량으로 업로드할 수 있나요?**

이 시점에서 파트너 센터 대시보드 및 파트너 센터 API는 Microsoft 클라우드 계약에 대 한 고객 승인 확인을 제공 하는 유일한 승인 된 수단입니다.

**새로 만들기 | 타사 플랫폼 또는 "미들웨어"를 사용 하는 파트너는 회사의 시스템을 파트너 센터에 연결 하는 기능을 수행 합니다. 무엇을 해야 하나요?**

파트너 센터 API를 통합 하는 플랫폼을 사용 하는 파트너는 고객 확인 프로세스를 자동화 하는 파트너 센터 API 기능을 통합 하는 데 필요한 계획 및 타임 라인과 관련 된 공급자와 직접 이야기 하는 것이 좋습니다.

**새로 만들기 | 고객의 전체 이름, 전자 메일 주소 및 Microsoft 클라우드 계약 승인 날짜를 가져오는 API 흐름 예제를 제공할 수 있나요?**

파트너는 파트너 센터 API 설명서에서 세부 정보 및 예제를 검토 하는 것이 좋습니다.

**새로 만들기 | 고객에 게 표시 하 고 API 프로세스의 일부로 확인 레코드에서 사용할 최신 Microsoft 클라우드 계약을 어떻게 할까요?**

현재 파트너 센터 API는 매우 기본적 이며 파트너가 Microsoft 클라우드 계약의 복사본을 프로그래밍 방식으로 검색 하는 것을 지원 하지 않으며, 확인의 일부로 고객이 승인한 Microsoft 클라우드 계약의 버전을 캡처하지 않습니다. 

**새로 만들기 | 이 프로세스에 파트너 센터 API를 사용 하려면 userID 매개 변수가 필요 합니다. Azure AD와 통합 되지 않는 타사 플랫폼을 통해 주문이 배치 되는 경우이 요구 사항을 관리할 어떻게 할까요? 있나요?**

이 경우 userID는 고객 동의 확인을 제공 하는 파트너 테 넌 트의 직원을 식별 합니다. 직원에 게 파트너의 테 넌 트 아래에 해당 하는 Azure AD 사용자 계정이 없는 경우 해당 목적으로 특정 사용자 계정을 지정 하 고 다음을 제공 하는 직원의 id를 별도로 캡처하는 프로세스를 만드는 것이 좋습니다. 알려준. 

**새로 만들기 | Microsoft에서 파트너를 고객 으로부터 직접 가져오는 대신 Microsoft 클라우드 계약에 대 한 동의를 확인 하는 데 필요한 이유는 무엇 인가요?**

고객이 직접 Microsoft가 아닌 CSP 파트너 로부터 구입 하 고 있습니다. Microsoft에서 제공 하는 서비스를 사용 하기 위해 Microsoft는 Microsoft 클라우드 계약에 설명 된 용어를 제공 합니다. Microsoft 클라우드 재판매인 규약을 사용 하려면 고객이 주문을 하기 전에 Microsoft 클라우드 규약을 검토 하 고 동의 해야 합니다. 

**새로 만들기 | API와 파트너 센터 대시보드 웹 사용자 인터페이스를 모두 사용 하 여 Microsoft 클라우드 계약 고객 동의를 확인 합니다. 2018 년 11 월 7 일 이후 고객 승인 확인을 위한 옵션은 무엇 인가요?**

두 옵션 모두 사용할 수 있으며 계약 고객 승인 Microsoft 클라우드 확인 하는 데 사용할 수 있습니다. 파트너 센터 대시보드를 사용 하는 트랜잭션은 2018 년 11 월 7 일부 터 확인 해야 합니다. 파트너 센터 API를 사용 하는 트랜잭션은 2019 년 3 월 22 일부 터 확인 해야 합니다.
